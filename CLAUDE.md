## Design Context

### Users
Primary users are hiring managers reviewing the portfolio to evaluate relevant technical work quickly. The interface should make it easy to scan projects, understand impact and stack, and identify fit for software/embedded roles.

### Brand Personality
Intelligent, Builder, Curious. The voice should feel clear, grounded, and technically credible while remaining approachable.

### Aesthetic Direction
Minimal and technical with dark mode as the default experience. Visual direction should be inspired by GitHub's clean, utilitarian style: restrained color use, strong content hierarchy, crisp spacing, and code-adjacent UI cues. Avoid decorative or playful visual treatments that weaken professional signal.

### Design Principles
1. Prioritize relevance and scanability for hiring managers with concise project framing and fast visual hierarchy.
2. Express technical credibility through clean structure, precise typography, and restrained, purposeful UI details.
3. Maintain approachability with plain language, readable spacing, and clear navigation.
4. Keep visuals minimal in dark mode, using color only to support hierarchy, state, and accessibility.
5. Favor practical interactions over novelty, with predictable behavior and reduced cognitive load.
